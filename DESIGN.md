# Hanauta-Chords 設計書

## 概要

鼻歌のメロディを録音・分析し、コード進行を自動で提案する Web ブラウザアプリ。

---

## 機能要件

### 1. 録音
- マイクから鼻歌を録音する
- 録音中はリアルタイムでピッチを検出する

### 2. メロディ解析 → 五線譜
- 録音した音声からピッチ（音程）とタイミングを抽出する
- 抽出した情報を音符列に変換する
- 音符列を五線譜として画面に表示する

### 3. コード生成
- 音符列からキー（調）を推定する
- 音楽理論のルールに基づき、複数のコードパターンを生成する
- 各パターンに印象説明を付与する

### 4. コードパターン表示
- 複数のコード進行パターンをタブ or カード形式で表示する
- 各パターンに印象説明（例：明るい、切ない、ジャジー）を表示する

## データフロー

```
[マイク入力]
    ↓ Web Audio API (AnalyserNode)
[リアルタイム ピッチ検出]
    ↓ pitchy (McLeod Pitch Method)
[周波数 → 音名変換]
    ↓ 例: 261.6 Hz → C4
[音符列の構築]
    ↓ 音名 + タイミング + 音価
[キー推定]
    ↓ 出現頻度・音程関係から調を判定
[コードパターン生成]
    ↓ 音楽理論ルール適用
[表示]
    ├─ VexFlow → 五線譜レンダリング
    └─ React UI → コードパターン一覧
```

## コード生成ロジック

### キー推定
- 音符の出現頻度をカウントする
- Krumhansl-Schmuckler アルゴリズム（音楽的音調プロファイルとの相関）でキーを判定する

### コードパターン（例：Cメジャーの場合）

| パターン名 | 進行 | 印象説明 |
|-----------|------|---------|
| 王道 | C → F → G → C | 明るく安定した、聴き慣れた雰囲気 |
| 切ない | Am → F → C → G | 感情的で切ない雰囲気 |
| ポップ | C → G → Am → F | キャッチーでポップな雰囲気 |
| ジャジー | Dm7 → G7 → Cmaj7 | 洗練された大人の雰囲気 |

### 対応スケール
- メジャーキー（長調）
- マイナーキー（短調、自然短音階）

## インフラ・実行環境

### コンテナ構成

```
[Podman]
    └─ ubi9/nodejs-20 (開発)
    └─ マルチステージビルド (本番)
           ├─ ステージ1: ubi9/nodejs-20 で npm run build
           └─ ステージ2: ubi9/nginx-120 で静的ファイル配信
```

### Dockerfile 方針
- ベースイメージ: `registry.access.redhat.com/ubi9/nodejs-20`
- 本番配信: `registry.access.redhat.com/ubi9/nginx-120`
- マルチステージビルドで本番イメージを軽量化する
- マイク使用のため HTTPS 必須（開発時は localhost で代替）

### 起動方法（予定）

```bash
# 開発
podman build -t hanauta-chords:dev --target dev .
podman run -p 5173:5173 hanauta-chords:dev

# 本番
podman build -t hanauta-chords:prod .
podman run -p 8080:8080 hanauta-chords:prod
```

---

## 実装方針

- **バックエンド不要**：すべてフロントエンドで完結する
- **外部 API 不使用**：コード生成は音楽理論ルールで実装する
- **プログレッシブ対応**：録音 → 解析 → 表示の順に段階的に動くようにする
- **コンテナファースト**：Podman + RHEL UBI での動作を前提とする
